PlasmaCodexBar — AI Usage Monitor

Track Your AI Subscription Quotas from the KDE Plasma Tray

PlasmaCodexBar is a native KDE Plasma 6 system tray widget that brings AI subscription quota monitoring directly to your desktop. Inspired by CodexBar for macOS, it provides real-time session and weekly usage tracking across six AI providers — with color-coded progress bars, expense estimates, and direct links to provider dashboards — all without leaving your workflow.

Supported AI Providers

Claude (Anthropic)

Full OAuth integration via Claude Code CLI. Tracks session usage (5-hour window with reset countdown), weekly quota, per-model breakdowns (Sonnet, Opus), and local expense estimates calculated from session JSONL files.

Codex / ChatGPT (OpenAI)

OAuth and API key support. Tracks credits balance, plan type (Free / Plus / Pro / Team / Enterprise), weekly usage, and unlimited status detection.

Gemini, Copilot, Cursor, OpenRouter

Adapter-based providers normalized to the same standard interface via the _AdapterBridge pattern, enabling future providers to be added with minimal code.

Widget Features

  • Robot icon in system tray — click to expand the full usage dashboard
  • Provider tabs — switch between Claude, Codex, Gemini, Copilot, Cursor, OpenRouter at a glance
  • Dual view modes — "Usage" (quota bars + reset countdowns) and "Stats" (pace, cost estimates) for Claude and Codex
  • Color-coded bars — green → yellow → red as quota fills up
  • Expense estimates — local token counting against current Anthropic/OpenAI pricing; no external calls
  • Adaptive polling — configurable refresh interval (default 5 minutes) with rate-limit awareness
  • Direct dashboard links — one click to the provider portal from within the widget

Technical Architecture

Python Backend

  • OAuth flows — Claude via ~/.claude/.credentials.json; Codex via ~/.codex/auth.json with 8-day token refresh cycle
  • Claude cost calculation — Opus 4.5 ($15/$75 per 1M tokens), Sonnet 4.5 ($3/$15); extracted directly from ~/.claude/projects/**/*.jsonl
  • ProviderStats dataclass — unified interface carrying utilization %, reset times, plan info, and pace metrics across all providers
  • Adapter pattern — _AdapterBridge normalizes external providers to the same dataclass; adding a new provider requires only an adapter
  • Dependencies: PyGObject ≥ 3.42, pycairo ≥ 1.20, GTK3 bindings — all system packages, no pip installs

QML Frontend

  • KDE Plasma 6 native — Repeater-based dynamic tab generation; dual compact icon / full widget representations
  • Configurable via KDE settings — enabled providers and refresh interval stored in ~/.config/ai-usage-monitor/main.xml (KConfigXT format)
  • Install methods: ./install.sh, KDE Store, or kpackagetool6 direct install

Tech Stack

Python 3.8+ QML KDE Plasma 6 OAuth 2.0 PyGObject GTK3 Bash MIT License

Available on GitHub and the KDE Store. Contributions and new provider adapters welcome.

PlasmaNGenuity — HyperX Mouse Configuration for Linux

Full Linux Replacement for HyperX NGenuity

PlasmaNGenuity is an open-source Linux substitute for HyperX's Windows-only NGenuity software. It provides native configuration of the HyperX Pulsefire Dart wireless gaming mouse through both a KDE Plasma 6 system tray widget and a standalone PyQt5 application — exposing every feature NGenuity offers, on Linux, without a VM.

Configuration Capabilities

DPI Profiles

Five independent DPI profiles with adjustable sensitivity from 50 to 16,000 DPI and per-profile color customization for the on-device LED indicator.

LED Effects

Static, breathing, spectrum, and trigger lighting modes for both the logo and scroll wheel, configurable per-profile.

Button Remapping

All six mouse buttons remappable to mouse functions, keyboard shortcuts, media controls, or DPI switching actions.

Macro Recording

Record custom key sequences with adjustable per-step delays and assign them to any button.

Polling Rate

Adjustable report rate from 125 Hz to 1000 Hz directly from the UI.

Battery & Device Info

Real-time battery percentage with color-coded tray indicator (green → yellow → orange → red), charging status, connection mode, firmware version, and voltage readings. Hotplug detection via udev for automatic device recognition.

Technical Architecture

HID Communication Layer

  • Supported devices: HyperX Pulsefire Dart wireless dongle (USB 0951:16E1) and wired/direct mode (USB 0951:16E2)
  • Protocol implementation: custom HyperX HID protocol in drivers/hyperx/protocol.py (15,698 bytes); covers full feature set reverse-engineered from the original software
  • hidapi for cross-platform HID device access; pyudev for hotplug detection
  • udev rules (99-plasmangenuity.rules) installed for non-root device access

Application Architecture

  • Layered design: Core manager → Driver interface → HyperX driver → HID provider / sysfs / UPower providers
  • Standalone app: PyQt5 system tray with left-click config panel, right-click context menu, animated charging indicator, and CLI tool with JSON output for scripting
  • Plasma widget: QML tray widget with real-time battery display and direct panel access — same feature set, native Plasma integration
  • UPower integration for system-level battery/power information alongside direct device reads

Distribution Options

  • Plasma widget: ./install-plasmoid.sh or pre-built .plasmoid package
  • Full app via pip: pip install "plasmangenuity[tray]"
  • Arch Linux / Manjaro: PKGBUILD included for pacman
  • Waybar module: shell script for Waybar status bar integration

Tech Stack

Python 3.8+ PyQt5 QML KDE Plasma 6 hidapi pyudev UPower HID Protocol udev MIT License

Open source and actively maintained. Issues and contributions welcome — especially support requests for other HyperX models.

Beating Blackjack — Statistical Analysis

Does Mathematical Strategy Actually Beat the House?

A data-driven investigation into whether basic strategy and card counting techniques produce a statistically significant advantage in blackjack. Built entirely in Jupyter Notebook with Python, this project quantifies the difference between random play, basic strategy, and card counting through extensive simulation and probability modeling.

What Was Investigated

Basic Strategy Analysis

Evaluation of standard blackjack basic strategy — the mathematically optimal play for every hand combination — and how much it reduces the house edge compared to random decisions.

Card Counting Investigation

Statistical assessment of card counting techniques, measuring the edge gained when tracking deck composition and adjusting bet sizing accordingly.

Probability Calculations

Precise probability modeling of winning odds under different rule sets, deck sizes, and player decisions — making the math behind every recommendation visible.

Comparative Simulation

Large-scale simulation runs comparing random play, basic strategy, and card counting head-to-head across thousands of hands to produce statistically reliable results.

Technical Approach

  • Platform: Jupyter Notebook — 98.6% of the codebase; Python for simulation logic
  • Methodology: Monte Carlo simulation over large hand samples to minimize variance and surface true expected values
  • Analysis scope: Basic strategy effectiveness, card counting advantage, probability-based decision trees, house edge quantification
  • Output: Charts and statistical summaries comparing strategies across thousands of simulated hands

Topics

Python Jupyter Notebook Statistics Probability Monte Carlo Data Analysis

SysLens — KDE Plasma System Monitor

Real-Time System Monitoring Widget for KDE Plasma 6

SysLens is a dark, futuristic system monitoring widget built natively for KDE Plasma 6. It delivers real-time visualization of seven subsystems — CPU, RAM, GPU, disk I/O, network, battery, and processes — through a compact, expandable panel popup. Zero external Python dependencies; all data is read straight from the Linux kernel via /proc and /sys.

CPU usage, temperature, power gauge & RAM breakdown with historical averages
GPU / VRAM metrics & disk I/O throughput
Network traffic history, battery health & top processes

Monitoring Sections

CPU

Donut usage chart, session-relative power gauge, temperature bar, and configurable short/long-window usage averages.

Memory & Swap

RAM donut chart, stacked breakdown bar (used / cache / free), swap tracking, and 1-day / 30-day historical averages.

GPU & VRAM

GPU usage metrics, VRAM secondary gauge, and historical tracking across configurable time windows.

Disk I/O

Live read/write line charts, throughput visualization, IOPS tracking, and root filesystem usage monitoring.

Network

Live upload/download speed donuts, traffic history, daily / weekly / monthly aggregates, and per-interface monitoring with virtual interface filtering.

Battery

Charge level, power status (charging / discharging), health percentage, cycle count, voltage and current readings.

Processes

Top CPU consumers with configurable list limit, real-time CPU and memory usage per process, live-sorted ranking.

Technical Architecture

Python Backend

  • Zero external dependencies — standard library only, no pip installs required
  • Direct kernel access — reads /proc/stat, /proc/meminfo, /proc/diskstats, /proc/net/dev, /sys/class/hwmon/, /sys/class/power_supply/ and more; no daemon required
  • Functional architecture — modular per-subsystem collection functions, no class hierarchy
  • Stateful persistence — usage averages survive widget restarts via ~/.local/state/syslens/state.json; boot ID tracking resets history on kernel reboots
  • Data pipeline — before-snapshot → CPU sampling window → after-snapshot → delta computation → historical aggregation → JSON output
  • Performance — history arrays capped at 48 items; configurable update interval from 1 to 10 seconds

QML Frontend

  • KDE Plasma 6 native — built with Kirigami 2 and Qt Controls, integrates as a panel popup widget via kpackagetool6
  • Reactive data flow — JSON from Python backend drives the UI through a single ingest() function keeping all panels in sync
  • Custom visualization components — DonutChart, LineChart, MarkerSummaryBar, RamBreakdownBar — all implemented from scratch in QML
  • Configurable — update interval, per-section visibility toggles, and process list limit all accessible from the right-click menu
  • Dark theme native — designed for Plasma's dark palette, no additional theming required

Visualization Components

  • DonutChart — arc gauge for percentage-based metrics (CPU, RAM, GPU, network speeds)
  • LineChart — time-series display for disk I/O and other rate-based data
  • MarkerSummaryBar — period comparison markers (short / long / all-time averages)
  • RamBreakdownBar — segmented horizontal bar showing used / cached / free memory split

Tech Stack

Python 3.9+ QML KDE Plasma 6 Kirigami 2 Qt Controls Linux /proc Linux /sys Bash
