🎓 VIMS4ALL - CampusBase School Management System

Enterprise School Management Platform

CampusBase (VIMS4ALL) is a comprehensive, full-stack school management system designed to streamline campus operations for educational institutions. Built with modern technologies and best practices, it provides a robust foundation for managing students, employees, courses, finances, and administrative tasks with multi-tenancy support.

🎯 Key Features

Student Management

Complete student lifecycle management including enrollment, status tracking, custodian information, and academic records. Automated PIN generation for unique student identification.

Employee & HR Management

Employee records, function assignments (director, registrar, instructor, accountant), career progression tracking, dependent management, and automated account closure on exit dates.

Academic Management

Course catalog, class scheduling, instructor assignments, academic term management, and comprehensive course registration system.

Finance & Accounting

Multi-account financial system with account types, ledger entries, transaction tracking, and role-based access for accountants and directors.

Multi-Tenancy Architecture

Institute-level data isolation ensuring complete separation between different schools/campuses. Automatic query scoping via custom middleware and managers.

Role-Based Access Control

Granular permissions system with superuser, institute admin, and employee roles. Function-based access control for specific operations (director, registrar, accountant, instructor).

🏗️ Technical Architecture

Backend (Django REST Framework)

  • Framework: Django 5.1 with Django REST Framework
  • API Documentation: Auto-generated OpenAPI schema via drf-spectacular with Swagger/ReDoc interfaces
  • Authentication: JWT-based authentication using SimpleJWT
  • Database: PostgreSQL with advanced query optimization and connection pooling
  • File Storage: MinIO (S3-compatible) for media and document management
  • Custom Managers: InstituteScopedManager and OptionallyScopedManager for automatic data filtering
  • Middleware: Custom InstituteContextMiddleware for request-level tenant isolation

Frontend (React + TypeScript)

  • Framework: React 18 with TypeScript for type-safe development
  • Build Tool: Vite for lightning-fast development and optimized production builds
  • UI Library: Material-UI (MUI) with custom theming and components
  • Routing: React Router v7 with role-based route protection
  • State Management: Context API for authentication and global state
  • Type Generation: Automated TypeScript type generation from OpenAPI schema
  • Data Grid: MUI X Data Grid for advanced table operations and data visualization

🛠️ DevOps & Infrastructure

Docker Docker Compose PostgreSQL MinIO Nginx Gunicorn Makefile

💼 My Development Process

  1. Requirements Analysis: Worked with client to understand complex educational institution workflows and data requirements
  2. Database Design: Designed normalized database schema with proper relationships and constraints for academic data
  3. API Architecture: Implemented RESTful API with comprehensive endpoints following OpenAPI standards
  4. Multi-Tenancy Implementation: Built custom middleware and managers for automatic institute-level data scoping
  5. Frontend Development: Created responsive, user-friendly interface with role-based views and permissions
  6. Type Safety: Implemented automated type generation pipeline ensuring frontend-backend consistency
  7. Testing & Deployment: Set up development and production environments with Docker containerization
